# Governance

The features will controll the how franz will act upon the actual kafka fleet. For instance, it is possible to manage how franz will act when creating a new topic in the real cluster or when it will update a existent topic config.


```edn
:governance {
    :topic [
        {
            ;; when a topic create/update happens, will check the replication factor
            ;; and reject if the replication factor is not 3
            :indicator :replication-factor
            :match (:not-equal 3)
            :is-mandatory? true
            :actions [
                :reject-topic-creation
                :reject-topic-update
            ]
        },
        {
            ;; when a topic create/update happens, will check the partitions
            ;; and reject if the partitions are bigger then 250
            :indicator :partitions
            :is-mandatory? false
            :match (:greater-than 250)
            :actions [
                :reject-topic-creation
                :reject-topic-update
            ]
        },
        {
            ;; when a topic create/update happens, will check the retention
            ;; and reject if the retention is bigger the 3 days
            :indicator :retention ;; it could be retention-ms, retention-minutes for more specific config
            :is-mandatory? false
            :match (:greater-or-equal-than "3d")
            :actions [
                :reject-topic-creation
            ]
        },
        {
            ;; do not allow unclean leader election
            :indicator :unclean.leader.election.enable
            :is-mandatory? true
            :match (:equal true)
            :actions [
                :reject-topic-creation
                :reject-topic-update
            ]  
        },
        {
            ;; this governance will act upon a indicator generated by Franz.
            ;; when replica size is greater or equal 100Gi it will increase
            ;; the partitions in 50%
            :indicator :replica-size
            :is-mandatory? false
            :match (:greater-or-equal-than "150Gi")
            :actions [
                :increase-topic-partitions
            ]
            :scale-config {
                :increase-percentage 50
            }
        },
        {
            ;; this governance will act upon a indicator generated by Franz.
            ;; when a retention is not used for 2d a period of time X
            ;; it will reduce the disk retention in 2d and 
            ;; add this retention period to the tiered storage
            :indicator :retention-not-used
            :is-mandatory? false
            :match (:greater-or-equal-than "2d"
                    :and
                    :for-period "90d")
            :actions [
                :move-retention-to-tiered-storage
            ]
        },
        {
            ;; the messages will be be lost if not consumed
            ;; if not consumed, the messages will be lost in 3d
            ;; so increase the retention in disk 50%
            ;; WARN: this has a risk of 
            :indicator :messages-will-be-lost-in
            :is-mandatory? false
            :match (:greater-or-equal-than "12h")
            :actions [
                :increase-retention
            ]
            :actions [
                :increase-percentage 50
            ]
        }
    ],
    :cluster [
    {
        :indicator  :avg-replica-per-broker
        :match (:greater-than 3000)
        :actions [
            :reject-topics-creation
            :reject-topics-replica-increase
        ]
    },
    {
        :indicator  :avg-leader-per-broker
        :match (:greater-than 3000)
        :actions [
            :reject-topics-creation
        ]
    }]
}

```
